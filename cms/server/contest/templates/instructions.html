{% trans %}Contest Rules{% endtrans %}

  1. {% trans %}This paper is only for invited Round 2 candidates. They will see the questions and submit programs on the saco-evaluator.{% endtrans %}
  2. {% trans %}You may code your solutions in C++, Java, Python3 or Pascal/Delphi. The time multipliers are{% endtrans %}:
    • C++: 1x
    • Pascal/Delphi: 1x
    • Java: 2x
    • Python: 10x
    {% trans %}For example, if the time limit given is 1 second, a Java solution may run for up to 2 seconds.{% endtrans %}
  3. {% trans %}Any attempt to access any other website or source of information will disqualify you, with the exception of online compilers such as{% endtrans %} rextester. {% trans %}For programming language documentation, refer to the {% endtrans %} {% trans %}Documentation{% endtrans %} {% trans %}link given on the pane on the left-hand side.{% endtrans %}
  4. {% trans %}Any submissions on the evaluator should NOT attempt any of the following:{% endtrans %}
    • {% trans %}access the internal evaluation server network,{% endtrans %}
    • {% trans %}fork or execute external programs,{% endtrans %}
    • {% trans %}create or access files,{% endtrans %}
    • {% trans %}attack the system security or the grader,{% endtrans %}
    • {% trans %}change file system permissions, or{% endtrans %}
    • {% trans %}read file system information.{% endtrans %}
    {% trans %}A competitor whose submissions attempt any of the above may be disqualified.{% endtrans %}
  5. {% trans %}Programs must pass the plagiarism check to be deemed valid.{% endtrans %}
  6. {% trans %}Log on using the username and password given to you by your teacher.{% endtrans %}
  7. {% trans %}There are five questions. There are 100 marks for each question. Each question is made up of subtasks. There are different marks for each subtask.{% endtrans %}
  8. {% trans %}For each subtask, your score will be the highest score obtained across all your submissions for that question.{% endtrans %}
  9. {% trans %}You will have two hours (120 minutes) to complete all 5 questions.{% endtrans %}
  10. {% trans %}You will only be permitted to submit at most 50 submissions for the 2 hours of the contest.{% endtrans %}
  11. {% trans %}You may assume that the user input will satisfy the problem specification and so you do not need to validate the input.{% endtrans %}
  12. {% trans %}You should not write code to produce only specific answers, as the judges will use other test cases.{% endtrans %}
  13. {% trans %}After 120 minutes, the server will end your participation. Should you finish before the time, check that you have answered all the questions.{% endtrans %}
  14. {% trans %}Do not wait until the last few minutes to submit your programs.{% endtrans %}
  15. {% trans %}Do not communicate the task statements or nature of the problems with other students from different schools until after the competition period is over.{% endtrans %}

{% trans %}Instructions{% endtrans %}

  1. {% trans %}Verify that your full name give on the bar in the top right hand corner is correct.{% endtrans %} {% trans %}If there are errors, then{% endtrans %} {% trans %}submit a question{% endtrans %} {% trans %}with the subject heading "Full Name", and in the body of the question, simply give your correct full name.{% endtrans %}
  2. {% trans %}Once you are ready to begin the contest, click on "Start". This sets the clock (top left) running. The Second Round questions will appear on the pane on the left-hand side. {% endtrans %}
  3. {% trans %}You will now have two hours (120 minutes) to complete all 4 questions.{% endtrans %}
  4. {% trans %}Click on "Statement", underneath the first question you wish to answer. Click on either "Statement in English" or "Statement in Afrikaans", depending on which language you wish to use.{% endtrans %}
  5. {% trans %}Using the template files, or otherwise, write a program to solve the problem. Cut and paste or type the program in the template provided for your language.{% endtrans %}
  6. {% trans %}Click on "Submissions". Click "Choose File", and select the source code of your program you wish to submit.{% endtrans %}
  7. {% trans %}Select the relevant programming language, and then click "Submit".{% endtrans %}
  8. {% trans %}You may use the{% endtrans %} {% trans %}Communication{% endtrans %} {% trans %}(top of left pane) but note that responses may be delayed.{% endtrans %}

{% trans %}Additional Important Notes{% endtrans %}

  1. {% trans %}Delphi/Pascal users may want to use the int64 datatype for numbers, to ensure you do not get integer overflow (e.g. on the evaluator, the 'integer' datatype is 16-bit).{% endtrans %}
  2. {% trans %}Your program should not output anything other than the answer. E.g. if you are asked to read in two integers A and B and print out their sum, you must print out ONLY the value of A+B. If your program prints out anything else, such as 'Enter A and B:', you will get ZERO.{% endtrans %}
  3. {% trans %}The 64-bit integer datatypes in each language (Python handles by default) are{% endtrans %}:
    • Java: long
    • C++: long long
    • Delphi/Pascal: int64
